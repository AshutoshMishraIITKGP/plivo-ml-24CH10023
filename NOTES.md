Our best configuration is a 4-layer, 192-dimension transformer with 6 attention heads (head_dim=32), utilizing SwiGLU activations, RMSNorm, and Rotary Position Embeddings (RoPE). This specific architecture was chosen because it perfectly maximizes the 2,000,000 parameter limit (reaching 1,967,808 parameters) while incorporating LLaMA-era efficiencies. Tying the input and output embeddings saved ~196k parameters, allowing us to allocate more capacity to the deep SwiGLU layers. We replaced standard GELU with SwiGLU (adjusted to 512 dimensions for exact parameter parity) because gating consistently improves autoregressive language modeling. RoPE replaces learned position embeddings, which not only saves parameters but significantly improves generalization across our 256-token context window. We process 8,192 tokens per step using gradient accumulation (batch size 32, sequence 256) to ensure the model sees enough data within our accelerated 500-step training cap. Finally, an AdamW optimizer paired with a 100-step linear warmup, gradient clipping (norm 1.0), and cosine decay to 1e-4 ensures stable gradients and prevents the initial loss spikes typical in heavily parameterized small models.
